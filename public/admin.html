Applications

Discord-based intake

Applicants use the in-Discord application panel (buttons + modal forms). Submissions are logged into your configured applications channel and stored by the bot.

Right now, review & decisions are handled via slash commands:

  • /app approve <user> division: Patrol / CID / SRT / etc.
  • /app deny <user> reason: <text>

We can extend this panel later to show a live list of pending applications and add Approve / Deny buttons that call the same bot logic.

Tickets

Support · IA · Training · Tech

The ticket system is fully Discord-native:

  • /ticket open type: general / ia / training / tech
  • Bot creates a private channel under the matching category.
  • Staff close with /ticket close, and the bot posts a text transcript to the log channel.

This view will later show a searchable archive of tickets and per-user histories once we wire it into the storage layer.

Reports & Requests

Roadmap

Report forms (citations, arrests, UOF, AAR, CID logs, TU shift summaries) and roster/role requests will live here as we connect the UI to the bot’s storage. The idea:

  • Discord-side buttons to submit reports & requests.
  • Entries appear here in an admin queue.
  • Each item gets a detail view and a “mark done” toggle.

Duty & Activity

On-duty board + stats

Duty tracking is currently driven by Discord commands and the on-duty board message:

  • /clock in assignment: Patrol / CID / Reaper / etc.
  • /clock out
  • /clock status – see your own current session.
  • /activity self – see your totals.
  • /activity member / /activity top – High Command views.

The dashboard can later be extended with live tables showing who’s on duty and time worked per assignment.
